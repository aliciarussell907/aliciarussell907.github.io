Code Log 1: HTML Semantic Tags

What is the difference between non-semantic and semantic tags in HTML?

Non-Semantic Tags
Tell us nothing about its content
Semantic Tags
Explicitly tell us about its content.
Some examples of non-semantic tags include:
  • <div> as it does not clearly describe what the content is but is rather used for styling and formatting.
  • <span> as it does not define the content, but it is also useful for styling
Some examples of semantic tags include:
  • <header> because it tells us the content is the header
  • <figcaption> because it tells us that the content is a caption for <figure> element

What are two (or more) benefits of using semantic tags?

Semantic tags are incredibly helpful because they help clue in on what the content is. Is this a header? A footer? A caption? An article? Without semantic tags it can be very difficult to understand what area of code you are working on or looking at. Another benefit is that semenatic tags improve accessiblity of the content. Screen readers and assistive technology rely on the semantic tags to understand how the content is structured.
